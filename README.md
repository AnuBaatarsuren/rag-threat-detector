# RAG Threat Detector

**LLM-based security log analysis using Retrieval-Augmented Generation**

Detects threats that rule-based systems miss by reasoning over behavioral patterns
using a combined MITRE ATT&CK + Sigma rules knowledge base and Qwen3 / DeepSeek / Claude.

---

## What it does

- Polls your ELK stack continuously for new security logs
- Groups logs by source IP into behavioral sessions (±10 minute windows)
- Retrieves relevant MITRE ATT&CK techniques and Sigma detection rules via RAG
- Compares session behavior against a learned per-IP baseline
- Asks an LLM to reason about whether the activity is malicious
- Writes enriched verdicts back to ELK with attack type, MITRE ID, confidence, and reasoning

## Why it's different from rule-based systems

Rule-based systems (Suricata, Kibana alert rules) detect threats by matching known signatures.
This system detects threats by reasoning over behavioral patterns — catching sub-threshold
reconnaissance, unknown C2 infrastructure, and multi-step attack chains that produce no
matching rule.

**Proven results on Linux-APT-2024 dataset:**

| Configuration | Precision | Recall | F1 |
|--------------|-----------|--------|----|
| No RAG (4B model) | 100% | 22% | 36% |
| RAG + 4B model | 95% | 77% | 85% |
| RAG + 14B + session | 66% | 100% | 79% |
| RAG + 14B + session + baseline | **100%** | **84%** | **91.8%** |

## Quick start

### 1. Clone and install

```bash
git clone https://github.com/yourusername/rag-threat-detector
cd rag-threat-detector
pip install -r requirements.txt
```

### 2. Build the knowledge base (MITRE ATT&CK + Sigma rules)

```bash
# Download MITRE ATT&CK JSON from https://attack.mitre.org/
# Save as mitre_attack.json in the project root

# Download Sigma rules
git clone https://github.com/SigmaHQ/sigma ~/datasets/sigma

# Build ChromaDB
python main.py --build-rag
```

### 3. Run setup wizard

```bash
python main.py --setup
```

The wizard asks:
- Which ELK stack to connect to
- Which LLM provider (Ollama local / DeepSeek API / Claude API)
- How long to use as baseline learning period

### 4. Start analysis

```bash
python main.py
```

## Research context

This system was developed as part of a thesis on LLM-based security log analysis.
The key research contributions are:

1. **Session-based behavioral grouping** — grouping logs by IP into time windows
   enables detection of multi-step attack patterns invisible to per-event analysis

2. **Baseline normality model** — per-IP behavioral statistics reduce false positives
   by giving the LLM context about what is normal for each source

3. **Combined MITRE + Sigma RAG** — augmenting abstract MITRE technique descriptions
   with concrete Sigma detection patterns improves LLM confidence calibration

4. **Cross-log-type generalization** — the same pipeline analyzes Linux, Windows,
   and firewall logs without retraining or modification
